Frequently Asked Questions (FAQ)

Getting Started

Q: What are the minimum system requirements for Lydian Agent?

A: For production deployment, we recommend: 16GB RAM, 4 CPU cores, 100GB storage. Docker 24.0+, PostgreSQL 15+, MongoDB 7+, Redis 7+, and Apache Kafka 3.0+.

Q: How long does implementation typically take?

A: Standard implementation: 2-4 weeks. This includes infrastructure setup, EHR integration, staff training, and pilot testing. Enterprise deployments with custom integrations may require 6-8 weeks.

Q: Do you offer cloud hosting or is it on-premises only?

A: We support both deployment models. Cloud: AWS, Azure, Google Cloud (fully managed). On-premises: Docker/Kubernetes deployment with your existing infrastructure. Hybrid deployments are also available.

Technical Questions

Q: How does Lydian Agent integrate with our existing EHR system?

A: We support HL7 FHIR R4, HL7 v2.x, and direct integrations with Epic, Cerner, MEDITECH, Allscripts, and Athenahealth. Our FHIR adapter translates data bidirectionally between your EHR and Lydian Agent.

Q: What happens if the quantum computing service is unavailable?

A: The Resource Optimization Agent automatically falls back to classical optimization algorithms (genetic algorithms, simulated annealing). While performance is reduced (65% vs 100% quantum efficiency), the system remains fully operational.

Q: Can we customize the AI agents for our specific workflows?

A: Yes. Enterprise customers can develop custom agents using our SDK. We also offer professional services to build specialized agents for unique clinical workflows, such as oncology treatment planning or surgical scheduling.

Security & Compliance

Q: Is Lydian Agent HIPAA compliant?

A: Yes. We maintain SOC 2 Type II certification, HITRUST CSF certification, and full HIPAA compliance. All PHI is encrypted at rest (AES-256) and in transit (TLS 1.3). We provide Business Associate Agreements (BAA) to all healthcare customers.

Q: Where is patient data stored?

A: Cloud deployments: US-only data centers with compliance certifications. Data never leaves your specified geographic region. On-premises: All data remains within your infrastructure. We never access patient data without explicit authorization and audit trails.

Q: How do you handle data breaches?

A: We maintain a comprehensive incident response plan with <4 hour response time. All security incidents are logged, investigated, and reported per HIPAA requirements. We carry $10M cybersecurity insurance.

Billing & Licensing

Troubleshooting Guides

Common Issues

Issue: API returning 401 Unauthorized errors

Solution:

  • Verify your access token hasn't expired (1-hour TTL)
  • Check that API key has correct scopes (clinical.read, clinical.write)
  • Ensure Authorization header format: "Bearer YOUR_TOKEN"
  • Regenerate API credentials if issue persists

Issue: Real-time monitoring not receiving updates

Solution:

  • Check WebSocket connection status (should be WSS, not WS)
  • Verify firewall allows outbound connections to wss://api.lydian-agent.com
  • Confirm patient IDs in subscription request are valid
  • Check browser console for connection errors

Issue: Quantum optimization taking longer than expected

Solution:

  • Check Quantum Computing service status at system health dashboard
  • Large optimization problems (>100 patients) may require 3-5 seconds
  • Consider using quantum simulator for testing (faster but less optimal)
  • Reduce QAOA circuit depth in config (trade accuracy for speed)
