Quick Start Guide

Get started with Lydian Agent in minutes. Our quantum-enhanced healthcare AI platform is designed for seamless integration with your existing infrastructure.

git clone https://github.com/lydian-agent/platform.git
cd platform
docker-compose up -d
python scripts/initialize.py --enable-quantum

Access the dashboard at http://localhost:8000

System Architecture

Lydian Agent leverages a microservices architecture optimized for healthcare operations:

  • FastAPI Backend: High-performance async REST API with automatic OpenAPI documentation
  • Multi-Database Layer: PostgreSQL for relational data, MongoDB for unstructured clinical notes, TimescaleDB for time-series vitals, Redis for real-time caching
  • Apache Kafka: Processes 10,000+ patient vitals per second with guaranteed delivery
  • IBM Quantum (Qiskit): QAOA algorithms for resource optimization and scheduling
  • Docker/Kubernetes: Production-ready containerization with horizontal scaling

API Authentication

All API requests require OAuth 2.0 authentication with PKCE flow for enhanced security:

POST /api/v1/auth/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET&
scope=clinical.read clinical.write quantum.access

Include the access token in all subsequent requests:

Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

Database Schema

Lydian Agent uses a hybrid database architecture:

  • PostgreSQL: Patient demographics, appointments, staff records, audit logs
  • MongoDB: Clinical notes, imaging metadata, unstructured EHR data
  • TimescaleDB: Vital signs time-series (10K writes/sec), continuous monitoring data
  • Redis: Session management, real-time agent communication, cache layer

Deployment Guide

Docker Deployment

docker build -t lydian-agent:latest .
docker run -d \
  -p 8000:8000 \
  -e DATABASE_URL=postgresql://user:pass@db:5432/lydian \
  -e KAFKA_BROKERS=kafka:9092 \
  -e QUANTUM_API_KEY=your_ibm_quantum_key \
  lydian-agent:latest

Kubernetes Deployment

kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/secrets.yaml
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl apply -f k8s/ingress.yaml

Configuration Reference

Environment variables for production deployment:

# Database Configuration
DATABASE_URL=postgresql://user:pass@localhost:5432/lydian
MONGODB_URI=mongodb://localhost:27017/lydian_clinical
TIMESCALEDB_URL=postgresql://user:pass@localhost:5432/timeseries
REDIS_URL=redis://localhost:6379/0

# Kafka Configuration
KAFKA_BROKERS=localhost:9092
KAFKA_TOPIC_VITALS=patient-vitals
KAFKA_TOPIC_ALERTS=clinical-alerts

# Quantum Computing
IBM_QUANTUM_TOKEN=your_token_here
QUANTUM_BACKEND=ibmq_qasm_simulator
QAOA_LAYERS=3

# Security
SECRET_KEY=your-secret-key-min-32-chars
ENCRYPTION_KEY=your-fernet-key
ENABLE_HIPAA_LOGGING=true
ENABLE_AUDIT_TRAIL=true

# AI Models
ENTERPRISE_AI_ENDPOINT=http://ai-model:8080
LLM_MODEL=advanced-medical-ai
MODEL_TEMPERATURE=0.3
MAX_TOKENS=2048

HIPAA Compliance Configuration

Enable comprehensive HIPAA compliance features:

# Enable encryption at rest
ENABLE_DATABASE_ENCRYPTION=true
ENCRYPTION_ALGORITHM=AES-256-GCM

# Enable encryption in transit
ENFORCE_TLS=true
MIN_TLS_VERSION=1.3

# Audit logging
AUDIT_LOG_RETENTION_DAYS=2555  # 7 years
ENABLE_PHI_ACCESS_LOGGING=true

# Access controls
ENABLE_RBAC=true
SESSION_TIMEOUT_MINUTES=15
REQUIRE_MFA=true
