Installation Guide

Prerequisites

  • Docker 24.0+ and Docker Compose 2.0+
  • Python 3.11+ (for local development)
  • Node.js 20+ (for frontend development)
  • PostgreSQL 15+, MongoDB 7+, Redis 7+
  • Quantum computing access (for quantum features)
  • Minimum 16GB RAM, 4 CPU cores

Quick Install with Docker

# Clone the repository
git clone https://github.com/lydian-agent/platform.git
cd platform

# Configure environment variables
cp .env.example .env
nano .env  # Configure your database credentials

# Start all services
docker-compose up -d

# Initialize database schema
docker-compose exec backend python scripts/init_db.py

# Create admin user
docker-compose exec backend python scripts/create_admin.py

# Access the platform at http://localhost:8000

Kubernetes Deployment

Deploy to Kubernetes Cluster

# Create namespace
kubectl create namespace lydian-agent

# Apply configurations
kubectl apply -f k8s/secrets.yaml
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/postgres-statefulset.yaml
kubectl apply -f k8s/mongodb-statefulset.yaml
kubectl apply -f k8s/redis-deployment.yaml
kubectl apply -f k8s/kafka-deployment.yaml
kubectl apply -f k8s/backend-deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl apply -f k8s/ingress.yaml

# Verify deployment
kubectl get pods -n lydian-agent
kubectl get services -n lydian-agent

# Scale backend pods
kubectl scale deployment/lydian-backend --replicas=3 -n lydian-agent

HIPAA Compliance Configuration

MEDIAN AGENT is designed to be HIPAA-compliant out of the box. Follow these steps to ensure full compliance:

1. Enable Encryption at Rest

# PostgreSQL encryption
ENABLE_DATABASE_ENCRYPTION=true
ENCRYPTION_ALGORITHM=AES-256-GCM
DATABASE_ENCRYPTION_KEY=your-32-byte-key-here

# MongoDB encryption
MONGODB_ENCRYPTION=true
MONGODB_KEY_FILE=/path/to/keyfile

2. Configure TLS/SSL

ENFORCE_TLS=true
MIN_TLS_VERSION=1.3
SSL_CERT_FILE=/path/to/cert.pem
SSL_KEY_FILE=/path/to/key.pem

3. Enable Audit Logging

ENABLE_AUDIT_TRAIL=true
AUDIT_LOG_RETENTION_DAYS=2555  # 7 years required
ENABLE_PHI_ACCESS_LOGGING=true
AUDIT_LOG_PATH=/var/log/lydian/audit.log

4. Configure Access Controls

ENABLE_RBAC=true
REQUIRE_MFA=true
SESSION_TIMEOUT_MINUTES=15
PASSWORD_MIN_LENGTH=12
PASSWORD_REQUIRE_SPECIAL=true
MAX_LOGIN_ATTEMPTS=3
ACCOUNT_LOCKOUT_DURATION_MINUTES=30

EHR Integration Guide

HL7 FHIR Integration

from lydian_agent import FHIRClient

# Initialize FHIR client
client = FHIRClient(
    base_url="https://fhir.ehr-system.com",
    auth_token="your-token"
)

# Fetch patient data
patient = client.get_patient("12345")

# Convert to MEDIAN AGENT format
lydian_patient = client.convert_to_lydian(patient)

# Submit to Clinical Decision Agent
from lydian_agent.agents import ClinicalDecisionAgent

agent = ClinicalDecisionAgent()
decision = agent.analyze(lydian_patient)

Custom Agent Development

Extend MEDIAN AGENT with your own custom AI agents:

from lydian_agent.core import BaseAgent
from lydian_agent.models import PatientData

class CustomRadiologyAgent(BaseAgent):
    """Custom agent for radiology analysis"""

    def __init__(self):
        super().__init__(
            name="Radiology Analysis Agent",
            version="1.0.0"
        )

    async def analyze(self, imaging_data: dict) -> dict:
        """Analyze radiology images"""

        # Your custom AI model logic here
        results = await self.ai_model.predict(imaging_data)

        return {
            "findings": results.findings,
            "severity": results.severity,
            "recommendations": results.recommendations,
            "confidence": results.confidence
        }

    async def generate_report(self, analysis: dict) -> str:
        """Generate structured radiology report"""
        return self.llm.generate(
            template="radiology_report",
            data=analysis
        )

# Register custom agent
from lydian_agent import register_agent

register_agent(CustomRadiologyAgent())

Performance Optimization

Database Optimization

# PostgreSQL connection pooling
DATABASE_POOL_SIZE=20
DATABASE_MAX_OVERFLOW=10

# TimescaleDB hypertable optimization
CREATE EXTENSION IF NOT EXISTS timescaledb;
SELECT create_hypertable('patient_vitals', 'timestamp');
SELECT add_retention_policy('patient_vitals', INTERVAL '90 days');

# Create indexes for common queries
CREATE INDEX idx_patient_vitals_patient_id
ON patient_vitals(patient_id, timestamp DESC);

CREATE INDEX idx_clinical_decisions_timestamp
ON clinical_decisions(created_at DESC);

Troubleshooting

Common Issues

Issue: Kafka connection timeout

# Check Kafka health
docker-compose logs kafka

# Increase timeout
KAFKA_CONNECTION_TIMEOUT_MS=30000

Issue: Quantum API errors

# Verify IBM Quantum credentials
python scripts/test_quantum_connection.py

# Use simulator for testing
QUANTUM_BACKEND=ibmq_qasm_simulator

Issue: Database migration failures

# Check migration status
python manage.py showmigrations

# Apply migrations manually
python manage.py migrate --fake-initial
